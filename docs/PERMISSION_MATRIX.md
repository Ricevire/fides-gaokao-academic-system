# 权限矩阵

本系统采用“角色 -> 能力点 -> 页面/接口”的权限模型。后端接口以能力点校验，前端页面和按钮也按同一套能力点渲染。

## 角色

| 角色 | 说明 |
| --- | --- |
| 管理员 `admin` | 系统管理、安全审计、账号发放、全部教务数据维护。 |
| 教务 `academic` | 维护教师、学生、班级、选科、排课、考试成绩等教务业务数据。 |
| 班主任 `head_teacher` | 兼容旧账号角色。新模型中班主任是教师业务职务，可与段长、任课老师等职务叠加。 |
| 教师 `teacher` | 教师登录身份。业务职务可叠加为段长、副段长、年级学科负责人、班主任、任课老师。 |
| 学生 `student` | 查看个人课表、考试和本人可见数据。 |

## 实现位置

- 后端矩阵：`src/permissions.js`
- 后端接口校验：`src/server.js` 的 `requirePermission(...)`
- 前端页面与按钮渲染：`public/app.js` 的 `can(...)`
- 数据范围过滤：`src/server.js` 的 `studentScope`、`classScope`、`teachingClassScope`、`timetableScope`、`examScoreScope`
- 组织隔离：登录 JWT 携带 `tenantId`、`campusId`、`academicYearId`；核心接口按 `tenant_id` 强制隔离，并对校区、学年提供筛选和写入上下文。
- 统一身份认证：OIDC 外部身份通过 `external_identities` 绑定本地账号，未绑定账号不能登录。
- 短信通知：账号发放短信由管理员触发，发送记录写入 `sms_messages`，只有管理员可检索。
- 数据分析：`/api/analytics/*` 使用 `analytics.read` 能力点，并继续按租户、校区、学年等上下文隔离数据。
- 教师职务：`teacher_duties` 支持段长、副段长、年级学科负责人、班主任、任课老师叠加；班主任和任课老师会同步到行政班、教学班，保证权限和排课数据一致。

## 验收重点

- 不同租户之间不能通过接口读取或修改对方数据。
- 校区、学年筛选必须只返回对应上下文内的教师、学生、班级、教学班、课表和考试数据。
- 非管理员不能访问 `/api/audit-logs`、`/api/audit-alerts`。
- 教务不能生成或重置教师、学生账号。
- 工作台分析接口不能跨租户读取成绩、学生或选科统计数据。
- 班主任只能维护本人负责行政班学生。
- 教师只能录入本人教学班学生的对应学科成绩。
- 学生不能看到教师管理、学生管理、行政班、选科组合、安全审计页面。
